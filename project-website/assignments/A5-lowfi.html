Assignment 5: Low-Fidelity Prototype

Early exploration of interaction concepts

Overview

Our low-fidelity prototype allowed us to quickly test core interaction concepts without investing in visual design. Using paper prototypes and basic wireframes, we validated our information architecture and key user flows with real users.

User Testing Insights

What Worked

  • Users easily understood the main navigation structure
  • Core task flow was intuitive and required minimal guidance
  • Information hierarchy made sense to users

What Needs Improvement

  • Some labels were confusing and need clarification
  • Users expected additional feature that we hadn't included
  • One interaction pattern didn't match user mental models

Key Learnings

  • Users prioritize [insight] over what we initially thought
  • The mental model for [feature] differs from our assumptions
  • Opportunity to simplify [specific element] in next iteration
